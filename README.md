# Cloud Security Architecture Compliance

This repository contains tools and documentation used to evaluate a healthcare organization's cloud security architecture and its alignment with HIPAA compliance requirements.

## Overview
In this project, we analyze the current cloud security practices of a healthcare organization transitioning to a hybrid cloud model. The goal is to identify gaps in security practices that could lead to non-compliance with HIPAA standards and recommend improvements.

### Tasks:
1. **Analyze Shared Responsibilities**: Define which security responsibilities are managed by the cloud provider and which are the responsibility of the healthcare organization.
2. **Evaluate Current Cloud Security Practices**: Assess adherence to cloud security best practices, identifying missing or incomplete practices.
3. **Compare Practices to HIPAA Compliance**: Map the organization’s current security practices to HIPAA requirements, identifying areas of non-compliance.
4. **Recommendations for Improvement**: Propose actionable recommendations to address security gaps and enhance compliance.

### Report Template
To document the findings, use the provided [report template](https://docs.google.com/document/d/1ZY5PdJTWUr4cBDgqsh5qomM8JvhZidLvaQefvysZKeE/copy).

### Standards
The report evaluates the organization against these key cloud security best practices:
- Data encryption at rest and in transit
- Centralized key management
- Multi-factor authentication (MFA)
- Principle of least privilege
- Automated monitoring, logging, and alerting
- Network segmentation
- Regular vulnerability assessments
- Documented and tested disaster recovery plans

### Report Overview:
The completed report outlines:
- **Shared Responsibility Analysis**: Identifying which responsibilities belong to the healthcare organization versus the cloud provider.
- **Cloud Security Architecture Evaluation**: A checklist to determine which best practices are implemented, missing, or incomplete.
- **HIPAA Compliance Comparison**: A mapping of the current security practices to HIPAA safeguards.
- **Recommendations for Improvement**: Specific actions to address identified security gaps and enhance HIPAA compliance.
